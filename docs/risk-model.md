# ScopeDiff Risk Model

ScopeDiff detects review-worthy capability changes. It does not prove malicious intent. Severity and confidence are conservative defaults that can be tuned later.

| ID   | Risk name                                                  | Default | Detection method                                                                                               | Example input                                  | Example finding                                | Why it matters                                    | False positives                           | Suggested review action                            | MVP                             |
| ---- | ---------------------------------------------------------- | ------- | -------------------------------------------------------------------------------------------------------------- | ---------------------------------------------- | ---------------------------------------------- | ------------------------------------------------- | ----------------------------------------- | -------------------------------------------------- | ------------------------------- | ----------------------------------------- | --- |
| R001 | MCP server added, removed, or modified                     | high    | Parse known MCP config files and compare `mcpServers` keys and values                                          | `{"mcpServers":{"github":{"command":"npx"}}}`  | `MCP server added: github`                     | Adds or changes tools an agent can call           | Legitimate setup change                   | Confirm server purpose, owner, and least privilege | yes                             |
| R002 | MCP command changed                                        | high    | Compare `command` field per server                                                                             | `command: "npx"` changed to `command: "bash"`  | `MCP command changed for github`               | Command choice controls local execution path      | Wrapper scripts may be normal             | Review executable and whether it is pinned         | yes                             |
| R003 | MCP args changed                                           | medium  | Compare `args` arrays as structured values                                                                     | `args: ["-y", "pkg"]`                          | `MCP args changed for filesystem`              | Args can expand paths, modes, or remote packages  | Benign version bump                       | Review new flags for access expansion              | yes                             |
| R004 | MCP env variable added                                     | medium  | Compare `env` keys per server                                                                                  | `env: {"GITHUB_TOKEN":"${GITHUB_TOKEN}"}`      | `MCP env added: GITHUB_TOKEN`                  | Tokens may grant external service access          | Placeholder examples are common           | Confirm token scope and need                       | yes                             |
| R005 | Credential-like variable name                              | high    | Match env keys and text tokens such as token, secret, key, credential, password, cookie, session               | `SLACK_BOT_TOKEN`                              | `Credential-like env var referenced`           | Indicates possible privilege or data boundary     | Names may be examples                     | Prefer least privilege and avoid real values       | yes                             |
| R006 | Executable command used                                    | medium  | Detect command names and shell words: npx, uvx, pipx, docker, curl, bash, sh, python, node                     | `command: "docker"`                            | `Executable command in agent config`           | Agents may trigger local execution                | Many MCP servers are command-based        | Review source, version, and arguments              | yes                             |
| R007 | Unpinned remote package                                    | high    | Detect `npx -y package`, `uvx package`, `pip install package`, docker `:latest`, missing version or digest     | `npx -y @scope/server`                         | `Remote package is not pinned`                 | Future installs may run different code            | Local package may be intentionally latest | Pin version, commit, or image digest               | yes                             |
| R008 | Remote URL added                                           | medium  | Diff added strings containing `https://`, `http://`, `git+ssh`, `ssh://`, `curl` URL                           | `curl https://example/install.sh`              | `Remote URL added in setup path`               | May create network dependency or download code    | Docs often link references                | Check whether URL is executed or just documented   | yes                             |
| R009 | High-risk Docker configuration                             | high    | Parse Dockerfile/Compose text for docker.sock, host mount, privileged, network host, user root                 | `privileged: true`                             | `Container runs privileged`                    | Breaks isolation expectations                     | Dev-only compose files may need it        | Document need, restrict scope, avoid host mounts   | yes                             |
| R010 | High-privilege AGENTS.md instruction                       | medium  | Text diff in instruction files for verbs: read, write, execute, network, publish, delete, commit, push, deploy | `You may push changes`                         | `Agent instruction adds push capability`       | Instructions influence agent behavior             | Natural language is ambiguous             | Confirm instruction is intended and bounded        | yes                             |
| R011 | High-privilege Cursor, Claude, Codex, or agent instruction | medium  | Scan `.cursor/rules`, skill `SKILL.md`, and tool-specific instruction files for capability verbs               | `Use shell to delete stale files`              | `Agent rule adds delete/execute guidance`      | Tool-specific instructions can widen behavior     | May describe prohibited actions           | Review context and add explicit limits             | yes                             |
| R012 | GitHub Actions permissions expanded                        | high    | Parse workflow YAML and compare `permissions` maps                                                             | `contents: write`                              | `Workflow permission expanded: contents write` | Increases token authority in CI                   | Required for release workflows            | Scope to job level and least privilege             | yes                             |
| R013 | Sensitive workflow trigger added                           | high    | Parse `on` keys for pull_request_target, workflow_run, repository_dispatch                                     | `on: pull_request_target`                      | `Sensitive trigger added`                      | Can run with elevated context or external input   | Maintainers may use safely with care      | Verify fork behavior and permissions               | yes                             |
| R014 | Workflow secrets usage added                               | high    | Detect added `${{ secrets.NAME }}` references                                                                  | `${{ secrets.NPM_TOKEN }}`                     | `New workflow secret used: NPM_TOKEN`          | Secrets can publish, deploy, or access services   | Legitimate deploy flow                    | Confirm event trigger and environment protection   | yes                             |
| R015 | External action not pinned to SHA                          | medium  | Parse `uses:` values and flag non-local actions without full commit SHA                                        | `uses: actions/checkout@v4`                    | `Action is not pinned to a commit SHA`         | Tags can move or be compromised                   | Many repos accept official tags           | Consider SHA pinning for sensitive jobs            | yes                             |
| R016 | Lifecycle script added                                     | high    | Compare `package.json` scripts for preinstall, install, postinstall, prepare, prepublishOnly                   | `"postinstall":"node setup.js"`                | `Lifecycle script added: postinstall`          | Runs during install and can surprise users/agents | Build tooling may require prepare         | Review script body and document purpose            | yes                             |
| R017 | Publish, deploy, upload, push operation added              | high    | Detect verbs/commands in workflows and scripts                                                                 | `npm publish`                                  | `Release operation added`                      | Can alter external systems or packages            | Normal release automation                 | Check permissions, tags, and environment gates     | yes                             |
| R018 | Permission-related config file changed                     | medium  | Flag changes to `.env.example`, settings, config, agent config files with scope-like keys                      | `SCOPE=repo:write`                             | `Permission-related config changed`            | May change default trust boundary                 | Examples may be harmless                  | Review whether defaults are safe                   | yes                             |
| R019 | Allowlist, denylist, sandbox, permission, or scope changed | high    | Keyword diff in config and docs                                                                                | `allow: ["*"]`                                 | `Allowlist appears broadened`                  | Guardrail changes can be more important than code | Context may be a test fixture             | Confirm before/after boundary                      | yes                             |
| R020 | Reduced review transparency                                | high    | Detect minified/encoded blobs, `curl                                                                           | bash`, compressed scripts, eval-like downloads | `curl ...                                      | bash`                                             | `Remote script is executed directly`      | Harder for reviewers to inspect behavior           | Some official installers use it | Prefer checked-in script or pinned digest | yes |

## MVP Rule Priority

MVP must implement R001, R002, R003, R004, R005, R006, R007, R009, R010, R011, R012, R013, R014, R015, R016, R017, R019, and R020. R008 and R018 are also in MVP but should use lower confidence when URLs or config values appear only in explanatory docs.

## Severity Adjustment Notes

- Increase one level when a finding combines secret access, shell execution, and network access.
- Increase one level when the finding affects CI, release, deployment, or fork-triggered workflows.
- Decrease confidence, not severity, when matching natural-language instructions.
- Do not mark all remote URLs high. Only execution or permission expansion should raise severity.
