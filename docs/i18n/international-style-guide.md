# International Messaging Guide

ScopeDiff should sound useful, transparent, and calm in every language. It should not use fear-based security marketing.

## Source Of Truth

- The English `README.md` is the source of truth for features, limitations, install commands, and release state.
- Localized pages are summaries for discovery and onboarding.
- If a localized page conflicts with the English README, fix the localized page.
- Shared visuals should stay synchronized with the English README. Current international visual: `docs/brand/international-readme-card.png`.

## Translation Rules

- Keep "review aid" language. Do not translate ScopeDiff into "full security audit" or "automatic protection".
- Keep core technical terms when the English term is common locally: MCP server, AI agent, GitHub Actions, token, CI, PR.
- Keep commands exactly as executable commands.
- Avoid exaggerated words such as "complete", "guaranteed", "attack-proof", "secure all", or their local equivalents.
- Mention false positives openly.
- Use a natural star CTA only when appropriate: if the project helps, a star helps others find it.
