# v0.1.0 Launch Plan

## Goal

Get real feedback from maintainers and AI-tool users who are already adding MCP servers, agent instructions, or GitHub Actions automation to repositories.

## Principles

- Be transparent that the author is sharing their own project.
- Ask for feedback, especially false positives.
- Do not ask people to star as a condition or reward.
- Do not post in unrelated communities.
- Do not mass DM.
- Do not automate comments, likes, reposts, follows, or submissions.

## Suggested Order

1. Publish npm package.
2. Create GitHub Release.
3. Verify `npx scopediff@latest --help`.
4. Update README if needed.
5. Share on personal X/LinkedIn accounts.
6. Submit Show HN if the project is stable enough for broad technical feedback.
7. Share to one relevant Reddit community only after checking rules.
8. Write Dev.to article.
9. Consider Product Hunt only after demo screenshots are polished.

## Feedback Channels

- GitHub Issues for bugs and false positives.
- GitHub Discussions only if enabled later.
- Replies on launch posts.

## Avoiding Spam

- Do not cross-post identical copy.
- Wait at least 24 hours between community posts.
- Tailor each post to the community.
- Skip communities where self-promotion is unclear or discouraged.
