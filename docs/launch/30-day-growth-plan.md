# 30-Day Compliant Growth Plan

## Week 1: Release and First Feedback

Day 1:

- Publish npm package and GitHub Release after approval.
- Verify install commands.
- Update published links log.

Day 2:

- Share personal X and LinkedIn posts.
- Ask for false-positive feedback.

Day 3:

- Create 2-3 first issues from local drafts after approval.
- Reply to any GitHub issues.

Day 4:

- Publish Dev.to technical article after approval.
- Add screenshots if available.

Day 5:

- Try Show HN after reading current HN guidelines.
- Do not repost if it gets no traction.

Day 6:

- Review feedback.
- Patch documentation confusion.

Day 7:

- Plan v0.1.1 based on real issues.

## Week 2: Better Examples

Day 8-10:

- Add more fixtures for real MCP config shapes.
- Improve false-positive docs.

Day 11:

- Share a short technical note about one finding type.

Day 12-14:

- Ask 3-5 peers for direct feedback. Do not ask them to star.
- Record feedback themes.

## Week 3: Community Fit

Day 15:

- Check Reddit community rules manually.

Day 16-17:

- Share in one relevant community if rules allow.

Day 18-21:

- Respond to comments with context and humility.
- Avoid arguing or reposting.

## Week 4: Consolidate

Day 22-24:

- Ship v0.1.1 if needed.

Day 25:

- Prepare Product Hunt only if screenshots and install path are solid.

Day 26-28:

- Submit to relevant newsletters or tool directories if they accept open-source developer tools.

Day 29:

- Review conversion data:
  - GitHub stars
  - npm downloads
  - Issues opened
  - False positives reported
  - README drop-off questions

Day 30:

- Publish a transparent progress update with lessons learned.
